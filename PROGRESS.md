# BULSA 개발 현황
> 작업 완료 시 이 파일만 업데이트. 내용은 항상 30줄 이내로 유지.

## 마지막 완료 작업
- 작업계획서 vs 작업 지시 구분 명확화:
  · GAME.state.workPlans[scenarioId] = { params, signedAt, signedBy } — 4시나리오 계획서 서명 시 자동 저장
  · 운전원 지시 cab 오버레이에 계획서 매개변수 + 확정 시각 + 작업반장명 표시
  · 계획서 미작성 시 "⚠ 운전원이 임의 판단 위험 ↑↑" 붉은 경고
  · 계획대로 진행 = 운전원 정상 작업 / 미작성·위반 = 사고 확률 ↑

## 다음 작업 (우선순위 순)
1. **안전관리자 적발 이벤트** + **과태료 시스템** — 잘 될 때는 무이슈, 종종 inspector NPC 출현 → 위반 항목 적발 + 과태료
2. **외부 GLB 에셋 통합** — assets/manifest.md 1순위(굴착기·펌프카·크레인·작업자) Quaternius/Kenney에서 다운로드 후 GLTFLoader 통합
3. **모션 시스템** — NPC 작업 애니메이션(굴착/타설/인양) + 캐릭터 GLB 스테이트머신
4. **이미 추가된 시스템 종합 검증** — 사용자 플레이 후 버그 리포트 기반 수정

## 디자인 원칙 (v2.0 방향)
- **행동 중심**: 팝업/클릭이 아니라 실제 도구 사용·측정·이동이 게임 결과에 영향
- **숙련도**: 도구별 사용 카운터 누적 → Lv 진행, 정확도/속도 보상

## 다음 작업 (우선순위 순)
1. **나머지 phase들 행동화** — 흙막이 점검(걸어다니며 H-pile 결속 확인), 안전난간 설치(드래그 배치), 신호수 배치 등
2. **다른 공정에 도구 시스템 확장** — 가스농도계(설비), 검전기(설비 LOTO), 줄자(기초 매트 면적 측정)
3. **공정 진행 흐름 검증** — 토공 미니게임 풀 동작 확인
4. **과태료/감독관 이벤트 시스템** — 작업계획서 누락 등 위반 적발

## 핵심 결정사항
- Phase 1(작업계획서)·Phase 2(안전검토)는 무사이 담당. BULSA v1.0은 Phase 3부터 시작
- 추가 시나리오(밀폐/전기/화재/비계/차량)는 v2.0 스텁으로 유지, 현재 삭제 불가
- Rapier.js 물리엔진은 v2.0으로 미룸
